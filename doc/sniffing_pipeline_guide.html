The 3 Scenarios

1
Ymir Server — Fully Automatic easiest

Launch the server, play, exit. The pipeline runs itself on shutdown.

bash ~/VoxCore/tools-dev/tc-packet-tools/start-worldserver.sh

That's it. When you type server exit in the worldserver console (or close the window), WPP + gap analysis + addon data + summary all run automatically. Output lands in C:\Users\atayl\VoxCore\out\build\x64-RelWithDebInfo\bin\RelWithDebInfo\PacketLog\

2
Standalone Retail Sniff — One Command most common

You have a .pkt file from a Ymir proxy capture of retail. Run from C:\Users\atayl\VoxCore\wago\

# cd to the wago directory first
cd ~/VoxCore/wago

# Full pipeline on your current sniff (39 MB, Mar 5 2026)
python3 process_sniff.py "C:/Users/atayl/VoxCore/ExtTools/ymir_retail_12.0.1.66263/dumps/dump_12.0.1.66263_2026-03-05_11-45-45.pkt"

# Same file, but generate SQL without applying (review first)
python3 process_sniff.py "C:/Users/atayl/VoxCore/ExtTools/ymir_retail_12.0.1.66263/dumps/dump_12.0.1.66263_2026-03-05_11-45-45.pkt" --no-import

# That file was already parsed by WPP — skip WPP, just run gap analysis
python3 process_sniff.py "C:/Users/atayl/VoxCore/ExtTools/ymir_retail_12.0.1.66263/dumps/dump_12.0.1.66263_2026-03-05_11-45-45.pkt" --no-wpp

# Show zone coverage after processing
python3 process_sniff.py "C:/Users/atayl/VoxCore/ExtTools/ymir_retail_12.0.1.66263/dumps/dump_12.0.1.66263_2026-03-05_11-45-45.pkt" --coverage
Tip: Future Ymir sniffs will also land in the ExtTools/ymir_retail_.../dumps/ folder. Just swap the filename.
3
Addon Data Only — No Sniff Needed quick

Just played retail with VoxSniffer running? Extract that addon data.

cd ~/VoxCore/wago
python3 process_sniff.py --addon-only

Auto-searches for VoxSnifferDB.lua in your WoW install's WTF folder.

Note: VoxSnifferDB.lua doesn't exist on your machine yet. You need to:
  1. Copy C:\Users\atayl\VoxCore\tools-dev\VoxSniffer\ into your retail WoW Interface\AddOns\ folder
  2. Enable it in the addon list, play for a while (visit NPCs, open vendors, accept quests)
  3. Log out — WoW writes SavedVariables on logout
  4. Then run --addon-only

WPP generates its own SQL files. These are not auto-imported — review them first.

Troubleshooting

ProblemFix
"Already processed (hash ...)"Add --force
"WPP not found"Check C:\Users\atayl\VoxCore\ExtTools\WowPacketParser\WowPacketParser.exe exists
"No parsed text found"WPP failed — check .NET 9.0 runtime
"VoxSnifferDB.lua not found"Install VoxSniffer addon, play retail
"pymysql not found"python3 -m pip install pymysql
"slpp not found"python3 -m pip install slpp
MySQL connection refusedStart MySQL (UniServerZ) first
High "Pokemon %" in WPPWoW build missing — run wpp-add-build.sh
Heads up: python3 and python resolve to different installs on this machine. Always use python3 — that's where pymysql and slpp are installed.
